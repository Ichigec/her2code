Проблема

❌ Агенты-одиночки: нет памяти, нет оркестрации

❌ Каждый раз — с нуля: контекст теряется между сессиями

❌ Нет стандарта качества: ошибки повторяются

❌ Развёртывание: сложно, долго, требует интернета

Решение

✅ Конвейер агентов: оркестратор → аналитика → разработка → аудит

✅ Память агента: навыки (skills) + граф знаний (Neo4j)

✅ Quality Gates: deny-by-default, 14 гейтов на 10 фаз

✅ USB-дистрибутив: dual-arch (ARM64+x64), offline

Идея: не «один агент для всего», а фабрика — получил задачу, создал агента под неё, исполнил, сохранил опыт для следующих

Приёмка

Как агент улучшает себя сейчас

1. Навыки (Skills) — после каждого сложного задания агент сохраняет успешный workflow как SKILL.md. Следующая сессия загружает его автоматически. Пример: «как санитизировать дистрибутив» → 14-страничный навык с pitfalls.

2. Наблюдатели (Observers) — 4 агента непрерывно анализируют работу: Auditor (качество процесса), Critic (переусложнение), Idea Generator (новые идеи), Knowledge Curator (сохранение в Neo4j).

3. Quality Gates — deny-by-default барьеры: bootstrap проверяет среду, pre-flight — контракты, security — SAST. Не прошёл → фаза блокирована.

4. AGENTS.md — единый стандарт: конвенции кода, pitfall'ы, цепочка эскалации. Каждый агент читает перед работой.

Цикл улучшения (Observe → Learn → Apply)

Observe: 4 наблюдателя мониторят каждую фазу

Curate: Knowledge Curator извлекает уроки → Neo4j-граф

Skill-ify: Успешный pattern → SKILL.md (процедурная память)

Gate-ify: Повторяющаяся ошибка → новый гейт в all_gates.yaml

Результат: 132 навыка накоплено. Каждый новый агент начинает не с нуля — он наследует опыт всех предыдущих.

Ключевая метафора

Не «один суперагент», а «завод»

Завод получает заказ (задачу) → собирает агента из компонентов (навыков) → запускает конвейер (пайплайн) → проверяет качество (гейты) → отправляет опыт в библиотеку (Neo4j) для будущих заказов.
