## Program pivot (July 2026)

Per Ботов's Habr article (3 days ago as of July 19, 2026):

- Old model: 4 fixed roles (ML Engineer, Data Analyst, Data Engineer, AI Product Engineer)
- New model: Agentic AI tracks, flexible trajectories
- "Подход устарел" — they explicitly abandoned the old role-based model
- Focus shift: from algorithms to **комплексные ИИ-продукты**
- "В 2026 году программа совершит переход от практико-ориентированного изучения алгоритмов к созданию комплексных ИИ-продуктов"

## JMLC evaluation rubric (5 criteria — the actual scoring grid)

Official rubric from ai.itmo.ru/junior_ml_contest (July 2026 snapshot). Map candidate experience to ALL FIVE — each must be visibly addressed in the letter AND the project pitch:

1. **Разработка и инженерия** — git, Docker, CI; MLOps/DevOps; качество кода; пайплайны ML; технологическая сложность решения; промышленная разработка.
2. **Data Science** — EDA; предобработка; выбор моделей; метрики; валидация результатов; понимание специфики данных.
3. **Применение ИИ** — AI-инструменты в работе над проектом; AI-агенты в задачах исследования, данных, разработки ПО, продукта.
4. **Продуктовое мышление** — проблема и ЦА; анализ конкурентов; продуктовые гипотезы; MVP; оценка импакта; обратная связь от пользователей и рынка.
5. **Мотивация** — цели в AI/ML; понимание ключевых особенностей программы; согласованность профессиональных/личных целей с обучением.

## Project types accepted by JMLC

Tell the candidate which bucket their project fits — it shapes the framing:
- Проект с работы (прикладная бизнес-задача, реальный бизнес-импакт)
- Проект с хакатона (победитель/призёр, доработанный после мероприятия)
- Выпускная работа (ВКР бакалавриата по ML)
- Научное исследование (публикации, препринты, конференции)
- Итоговый проект с курсов AITH (ML Engineering, LLM-инженерия)
- AI-стартап (MVP, метрики, обратная связь)
- AI в образовании (EdTech: адаптивное обучение, автопроверка, персонализация)

Team projects: each member submits separately and describes individual contribution.

## Admission paths

### Junior ML Contest (JMLC)
- Project-based admission WITHOUT entrance exams
- Submit: CV, motivation letter, code repo link, project description (≤5 pages), presentation
- Three waves per year
- Project types accepted: production solutions, research, thesis, startups, hackathon results
- Evaluation criteria: model selection, experiments, MLOps, presentation, code quality

### Portfolio Contest
- Three sections: motivation letter (25 points max), CV, academic achievements (25 points)
- 3 stages: tests → soft skills assessment → interview
- Motivation letter: 1 A4 page, why this program and why ITMO

### Standard entrance exams
- Tests + interview
- 3 stages: test tasks → soft skills (talent-tracker.tech) → interview + soft skills assessment
