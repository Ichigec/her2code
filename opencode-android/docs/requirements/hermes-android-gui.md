# Requirements: Hermes Android GUI
**Date:** 2026-06-12  
**Status:** Draft  
**Project:** Android-клиент для общения с Hermes Agent и OpenCode+

---

## 1. Executive Summary

Разработать нативное Android-приложение (Kotlin + Jetpack Compose) для полноценного взаимодействия с AI-агентом Hermes (и OpenCode+) через OpenAI-совместимый API. Приложение должно обеспечить: чат со стримингом, переключение моделей и агентов, локальное сохранение диалогов, настройки подключения, управление инструментами (tools/MCP), а также возможность исполнения кода на хосте.

---

## 2. Actors

| Actor | Description |
|-------|-------------|
| **Пользователь** | Владелец Android-устройства, взаимодействует с AI через чат |
| **Hermes API Server** | Бэкенд Hermes Agent, OpenAI-совместимый эндпоинт (`http://host:8642/v1`) |
| **OpenCode+ API** | Альтернативный бэкенд OpenCode+ (OpenAI-совместимый) |

---

## 3. Functional Requirements (Use Cases)

### UC-1: Отправка сообщения и получение ответа
- Пользователь вводит текст в поле ввода
- Сообщение отправляется на API (`POST /v1/chat/completions`)
- Ответ отображается в чате в реальном времени (streaming SSE)
- Markdown-рендеринг сообщений (код, списки, жирный, ссылки)
- Возможность прервать генерацию (кнопка Stop)

### UC-2: Переключение моделей
- Загрузка списка моделей с API (`GET /v1/models`)
- Отображение списка в выпадающем меню / диалоге
- Выбор активной модели, сохранение выбора локально
- Индикатор текущей модели в UI

### UC-3: Переключение агентов / персон
- Список доступных агентов (из конфигурации Hermes: `agent.personalities`)
- Ручное добавление кастомных агентов (system prompt)
- Переключение между агентами в UI
- Возможность задать произвольный system prompt

### UC-4: Сохранение и управление диалогами
- Автосохранение всех сообщений в локальную БД (Room/SQLite)
- Создание новых диалогов
- Список диалогов с превью последнего сообщения
- Удаление диалогов
- Поиск по диалогам

### UC-5: Настройки приложения
- URL API-сервера (по умолчанию `http://192.168.x.x:8642/v1`)
- API-ключ
- Выбор темы (светлая/тёмная/системная)
- Размер шрифта
- Настройки стриминга (вкл/выкл)
- Таймаут запросов
- Экспорт/импорт настроек

### UC-6: Исполнение кода на хосте
- Поддержка tool `terminal` — отправка команд на выполнение
- Запрос подтверждения у пользователя перед выполнением
- Отображение результата выполнения (stdout/stderr, exit code)
- История выполненных команд
- Настройка sandbox (max timeout, allowed paths, etc.)

### UC-7: Управление Tools и MCP
- Просмотр списка доступных tools/MCP (локальная конфигурация)
- Включение/отключение отдельных tools
- Включение/отключение MCP-серверов
- Сохранение конфигурации tools в настройках
- Отправка списка активных tools в API-запросах

### UC-8: Навигация и UX
- Bottom navigation: Чаты, Настройки
- Анимированные переходы
- Pull-to-refresh списка диалогов
- Жесты: свайп для удаления диалога
- Toast/снэкбар для уведомлений
- Обработка ошибок сети с повтором

## 4. Non-Functional Requirements

| NFR | Target |
|-----|--------|
| **Platform** | Android 8.0+ (API 26+), Kotlin 1.9+, Compose BOM 2024 |
| **Performance** | Рендеринг чата с 500+ сообщениями без лагов (lazy list) |
| **Network** | Поддержка HTTP/HTTPS, SSE streaming, retry с exponential backoff |
| **Security** | API-ключ хранится в EncryptedSharedPreferences; терминал — sandbox |
| **Storage** | Room DB для диалогов, DataStore для настроек |
| **UI** | Material 3, адаптивная вёрстка (phone + tablet), тёмная тема |
| **Accessibility** | Content descriptions, минимальный таргет 48dp, TalkBack |
| **Offline** | Просмотр сохранённых диалогов без сети |
| **Size** | APK < 30 MB |

---

## 5. Constraints

- **Технологический стек:** Kotlin, Jetpack Compose, Room, Retrofit, OkHttp, Hilt, DataStore
- **Архитектура:** MVVM + Clean Architecture (data/domain/presentation)
- **API:** OpenAI Chat Completions API (совместимость с Hermes и OpenCode+)
- **Язык интерфейса:** Русский (с возможностью расширения на EN)
- **Среда разработки:** Android Studio + Gradle KTS

---

## 6. Out of Scope

- Голосовой ввод / TTS (может быть добавлен позже)
- Шифрование сообщений end-to-end
- Поддержка нескольких аккаунтов одновременно
- Виджеты на рабочий стол
- Поддержка iOS
- Встроенный file browser (только через tools)

---

## 7. Acceptance Criteria

1. Приложение запускается на Android 8.0+ без крашей
2. Пользователь может отправить сообщение и получить потоковый ответ
3. Пользователь может переключать модели и агентов
4. Диалоги сохраняются при перезапуске приложения
5. Настройки (URL, ключ) применяются без перезапуска
6. Команды терминала выполняются после подтверждения пользователем
7. Tools можно включать/отключать в настройках
8. Тёмная тема работает корректно на всех экранах
