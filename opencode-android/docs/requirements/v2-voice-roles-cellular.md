## Фича 1: Голосовой онлайн-чат (Real-time Voice-to-Voice)

### Ключевые решения
- **Тип:** Real-time voice-to-voice (звонок) — говоришь → сразу слышишь ответ голосом, без текстового посредника
- **STT:** Hermes API (faster-whisper на ПК) — отправляем аудио-поток на сервер, он возвращает текст
- **TTS:** Hermes API (Edge TTS / ElevenLabs / OpenAI — через сервер)
- **Аудио-формат:** Opus для экономии трафика (сотовой сети)
- **Push-to-talk:** зажал микрофон → говорит, отпустил → отправил + сразу слушает ответ

### Интерфейс
- Кнопка микрофона в ChatInputBar (рядом с полем ввода)
- Во время разговора — waveform/индикатор громкости
- Во время ответа — анимация «говорящего» ИИ
- Вибрация при начале записи
- Параллельно показывать текст (расшифровку + ответ)

### Технические детали
- Аудио-поток: WebSocket или chunked HTTP к Hermes API
- Формат: Opus 16kHz mono
- Задержка: <500ms на chunk STT
- TTS стриминг: начинать озвучку до полной генерации ответа

## Фича 2: Убрать OC+, добавить персоны/роли

### Ключевые решения
- **OC+ полностью удалён:** переключатель H/OC+ из BottomToolbar, поля openCodeApiUrl/openCodeApiKey из Settings
- **Персоны:** выпадающий список (dropdown) по нажатию на иконку в нижней панели
- **15 персон** из `config.yaml` Hermes: technical, concise, creative, helpful, teacher, philosopher, noir, shakespeare, pirate, surfer, catgirl, kawaii, uwu, hype (+ default)
- При выборе персоны — её system prompt отправляется в API
- System prompt персоны заменяет/дополняет системный промпт агента

### UI
- Вместо пары чипов [H] [OC+] — одна иконка «маска/роль» → раскрывается dropdown
- Каждая персона: иконка-эмодзи + название
- Сохранять выбранную персону в настройках
- Отображать выбранную персону в BottomToolbar (краткое имя/иконка)

## Фича 3: Подключение по сотовой сети (Multi-URL + Tailscale)

### Ключевые решения
- **Основной канал:** Wi-Fi → прямой IP (<YOUR_LOCAL_IP>:8643)
- **Fallback канал:** Tailscale VPN → mesh-IP (100.x.y.z:8643)
- **Архитектура:** Multi-URL с авто-переключением
- **Приоритет:** сначала прямое соединение (быстрее), при недоступности → Tailscale

### Технические детали
- Настройки: Primary URL + Fallback URL (вместо одного поля)
- Health-check: пинговать `/health` на primary URL; если 3 раза fail → переключиться на fallback
- Индикация в UI: зелёный (прямое) / синий (туннель) / красный (нет связи)
- Авто-возврат на primary при его доступности (периодический health-check раз в 30 сек)
- Tailscale: установить на ПК, получить mesh-IP, вписать в настройки
- На ПК: `tailscale up` + проброс порта через Tailscale Serve или прямой mesh-IP

### Безопасность
- Tailscale = WireGuard E2E-шифрование
- Никаких открытых портов в интернет
- Hermes API остаётся закрытым (только внутри Tailscale-сети)
